## API Documentation

Once running, visit `http://localhost:8000/docs` for interactive API documentation (auto-generated by FastAPI).

### Endpoints Summary

1. **POST /api/games** - Create new game
2. **GET /api/games/{id}** - Get game state
3. **POST /api/games/{id}/roll** - Roll 4 dice
4. **POST /api/games/{id}/choose** - Choose pairing (0-2)
5. **POST /api/games/{id}/stop** - Stop turn, save progress

## How It Works

### Backend Flow
```
1. Client requests new game
2. Server creates GameState with empty boards
3. Client rolls dice
4. Server generates all 3 pairings
5. Server filters to valid pairings only
6. Client selects valid pairing index
7. Server applies pairing to temp progress
8. Client rolls again OR stops
9. On stop: commit temp â†’ permanent
10. Check win condition (3 columns)
```

### Frontend Flow
```
1. App loads, creates game via API
2. Render board with current state
3. User clicks "Roll Dice"
4. Animate dice, show pairings
5. User selects pairing (highlights)
6. User confirms (animates movement)
7. State updates, re-render board
8. User rolls again or stops
9. On bust: clear temp, switch player
10. On win: show game over screen
```

## Key Game Rules Implemented

1. **Dice Pairing**: 4 dice â†’ 3 unique ways to pair
   ```
   [A, B, C, D] â†’
     (A+B, C+D)
     (A+C, B+D)
     (A+D, B+C)
   ```

2. **Runner Limit**: Maximum 3 active columns at once

3. **Completed Columns**: Can't be used by either player

4. **Bust Condition**: No valid pairings = lose all temp progress

5. **Win Condition**: First to complete 3 columns

6. **Column Heights**:
   - Extremes (2, 12): 3 steps
   - Outer (3, 11): 5 steps
   - Inner (4, 10): 7 steps
   - Middle (5, 9): 9 steps
   - Center (6, 8): 11 steps
   - Peak (7): 13 steps (longest)

## Development Decisions

### Why FastAPI?
- Modern Python async framework
- Auto-generated API docs
- Built-in validation with Pydantic
- Easy CORS setup
- Fast performance

### Why React + Vite?
- Component-based UI
- Fast development with HMR
- Vite is faster than Create React App
- No complex build configuration needed

### Why Framer Motion?
- Declarative animations
- Easy spring physics
- Layout animations
- AnimatePresence for enter/exit
- Better than CSS-only for complex animations

### Why Pure CSS?
- No build step for styles
- Modern CSS has everything needed
- Gradients, animations, grid, flexbox
- Faster than Tailwind for this scale
- Easier to customize
